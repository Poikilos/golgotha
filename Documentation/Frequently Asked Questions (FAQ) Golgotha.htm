Frequently Asked Question (FAQ)
Q: Win95/NT : Running i4_make reports error saying bad command/file not found, what gives?
A: First if you have a windows directory other than c:\windows, you should reflect this in the i4_make.cc file and rebuild i4_make by running build_i4_make.bat. Second if i4_make tries to execute a command with a space in the filename, it will probably not work. Don't extract the source to a directory such as "Program Files/golgotha". To see the command i4_make is having a problem with, run "i4_make -v". "i4_make -?" will report other possibly useful switches.

Q: What does it mean when it says "golgotha.cd" is missing?
A: Golgotha.cd is like a doom WAD file or a ZIP/TAR file. It can store a bunch of files in one file. This reduces the clutter for the end user. However during development it is a pain to have to rebuild this file everytime you run the game. So when golgotha opens a file it looks inside the golgotha.cd file first, if it is not found it will read the file from your hard drive. In this release we do not have a golgotha.cd file, so all files are read from your hard drive. You can use the cd_maker program (included) to build a golgotha.cd file. If you do, you may wish to use the command line option "-netfirst" when running golgotha. This causes golgotha to first read from your hard drive, then from golgotha.cd if the file is missing.

Q: I want to use Golgotha's rendering code in my game, where do I start?
A: Checkout the test/spin_cube program. It renders a spinning cube with a texture applied and it pretty heavily commented.
